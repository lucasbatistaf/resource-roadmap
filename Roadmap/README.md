## **Primary Skills**

#### Primary skills are *FUNDAMENTAL* to your development, don't neglect them!

This list is in order of importance

### SQL
*The foundational language for querying, transforming, and manipulating data in relational databases and modern data warehouses. Data and analytics engineers use SQL daily to build pipelines, write transformations, and answer business questions. Mastery includes window functions, CTEs, query optimization, and understanding how different warehouse engines (BigQuery, Snowflake, Redshift) execute SQL.*

### Data Storage
*Understanding where and how data lives across its lifecycle. Covers relational databases (PostgreSQL, MySQL), cloud data warehouses (BigQuery, Snowflake, Redshift), data lakes (S3, GCS, ADLS), and lakehouses (Delta Lake, Apache Iceberg). Engineers need to know the tradeoffs between storage formats (Parquet, ORC, CSV), partitioning strategies, and when to use which storage layer for different workloads.*

### Data Modeling
*The practice of structuring and organizing data so it is accurate, performant, and easy to consume by analysts and BI tools. Includes dimensional modeling concepts (star and snowflake schemas, facts and dimensions), normalization vs. denormalization tradeoffs, and modern layered approaches like the medallion architecture (bronze/silver/gold). A good data model is the difference between a data platform people trust and one they work around.*

### Data Quality
*Ensuring that data is accurate, complete, consistent, and timely — because bad data leads to bad decisions. Covers writing data tests (uniqueness, not-null, referential integrity, custom business logic), setting up anomaly detection, monitoring freshness, and building alerting systems. Tools like dbt tests, Great Expectations, and Soda are commonly used. Data quality is both a technical and cultural discipline.*

### Cloud Basics
*Modern data infrastructure lives in the cloud, making this a non-negotiable foundation. Covers core concepts across AWS, GCP, and Azure — including compute (VMs, serverless functions), object storage (S3, GCS), IAM and permissions, networking basics, and managed data services. Engineers don't need to be cloud architects, but should be comfortable navigating the console, managing credentials, and understanding how cloud costs relate to data workloads.*

### AI Fluency
*Understanding how to work effectively alongside AI tools and integrate them into data workflows. Includes using LLMs to assist with SQL generation, code review, and documentation, as well as knowing how to evaluate AI outputs critically. At a deeper level, covers awareness of ML pipelines, feature stores, and how data engineering supports model training and serving. AI fluency is increasingly what separates engineers who scale their impact from those who don't.*
